# BrightQoL

A large-scale, modular Lua scripting project comprising **350+ files** and **30+ independent modules**, built from the ground up with a focus on clean architecture, performance, and maintainability.

## Overview

BrightQoL is a complex event-driven application written entirely in Lua. It demonstrates real-world scripting skills including modular system design, UI programming, state management, and dynamic configuration handling.

The project was developed and maintained solo — from architecture decisions to implementation, testing, and iterative improvement.

## Technical Highlights

- **350+ Lua files** organized across a clean modular structure
- **30+ independent modules**, each self-contained with its own configuration and lifecycle
- **Event-driven architecture** — modules react to system events without tight coupling
- **Dynamic UI system** — programmatic creation and management of interface elements
- **Persistent configuration** — per-module settings with import/export support
- **Runtime toggling** — modules can be enabled/disabled independently without restart
- **Slash command system** — full CLI interface for runtime control
